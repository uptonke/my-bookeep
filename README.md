# 個人記帳系統

## 使用方式

如果是全新資料庫：

1. 到後端資料庫的資料庫編輯器執行 `schema.sql`。
2. 把 `index.html`、`style.css`、`script.js`、`config.js` 上傳到網站專案。
3. 開啟網頁後測試新增年度、帳戶、分類、預算項目與交易。

如果你已經執行過舊版資料庫結構：

1. 不要重跑 `schema.sql`。
2. 如果還沒升級到退款與訂閱版，先執行 `migration_v2_refund_subscription.sql`。
3. 如果已經是 v2，再執行 `migration_v3_remove_receipt_attachment.sql`。
4. 再上傳新版前端檔案。

## 這版新增什麼

1. 流水帳新增「退款」類型。
2. 退款可以關聯原本那筆支出。
3. 預算與分類報表改成看「淨支出」，退款會扣回支出。
4. 「定期交易」改成「訂閱管理」。
5. 訂閱管理改成服務名稱、付款週期、下次扣款日、使用中或已取消。
6. 刪除交易表單中的收據連結與附件連結。

## 退款怎麼記

不要刪原本支出。新增一筆「退款」：

- 類型：退款
- 帳戶：退款回來的帳戶
- 分類：跟原支出相同
- 預算項目：跟原支出相同
- 關聯原支出：選原本那筆消費


## 顏色

分類與標籤的顏色已改成中文選單，不需要手動輸入色號。

## 注意

這是單人測試版，沒有登入驗證與列層級資料隔離。不要放敏感真實帳務。


## v4 更新

- 刪除信用卡「回饋類型 / 回饋率」欄位，避免假精準並簡化程式碼。
- 新增信用卡時，「對應帳戶」只會列出帳戶類型為「信用卡」的帳戶。
- 儲存資料時，沒有 id 的新資料改用 insert，有 id 的既有資料才用 upsert；降低新增訂閱後列表不更新的機率。
- 新增資料成功後會顯示「已儲存」。

若你已經執行 v3，請只執行 `migration_v4_remove_credit_rewards.sql`。


## v5 修正

- 修正「訂閱管理」顯示已儲存但列表仍空白的前端讀取問題。
- 儲存訂閱後會先把新資料直接放進列表，再重新讀取資料庫。
- 讀取資料改成部分失敗不阻斷全部頁面，避免某個報表 view 異常導致訂閱列表無法更新。
- v4 升 v5 不需要執行新的 SQL migration，只要覆蓋前端檔案。
