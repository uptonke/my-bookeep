# 個人記帳系統 v14

## 更新方式

1. 覆蓋 GitHub repo 根目錄的 `index.html`、`style.css`、`script.js`、`config.js`。
2. 開啟網站時使用 `?v=13`，確認左側顯示「雲端資料庫｜v14」。
3. Supabase 權限只需要執行一次 `final_permissions.sql`。

## v14 修正

- 修正年度預算、流水帳等表單出現「未知表單：[object HTMLInputElement]」。
- 修正原因：表單內有 `name="id"` 的隱藏欄位，導致 `form.id` 被 HTMLInputElement 覆蓋。現在全部改用 `form.getAttribute("id")`。
- 修正訂閱表單不應同時進入通用儲存流程。
- 刪除流程改成刪除後再查一次資料庫確認，不再依賴 Supabase 是否回傳被刪資料。
- 權限 SQL 收斂成單一 `final_permissions.sql`。

v35：新增提撥型預算計算。當預算項目設定為「期間：每月 / 每週」且「結轉模式：餘額結轉」時，系統會用「每次提撥金額 × 已提撥次數 − 已花費」計算目前可用額度。固定預算維持原本邏輯。無須改資料庫。


v36：年度預算月提撥化。年度設定新增「預算模式、每月預算、計算起點」。選「月提撥」時採模式 B：從記帳開始月算已提撥次數；目前可用預算 = 每月預算 × 已提撥月份 + 前期結轉 − 已花費。既有專案請先執行 `migration_v36_monthly_year_budget.sql`。


v37：年度結轉改為「自訂結轉金額」，不再固定使用剩餘預算；點「自訂結轉到下一年」可輸入任意金額。預算項目提撥計算改以「提撥次數」為核心，並新增提撥起始日 / 結束日欄位。無須新增資料表。


v38：修正 v37 年度結轉邏輯。結轉金額不再讓使用者自訂，而是固定使用今年剩餘額度；使用者自訂的是「下一年新增預算 / 下一年每次提撥金額」。也修正文案，明確使用「每次提撥 × 已提撥次數」。


v39：新增手動預算提撥紀錄。新增 `budget_contributions` 資料表與「預算提撥」表單；提撥型預算不再用自動推估次數，而是用實際提撥紀錄累積金額計算可用額度。既有專案請先執行 `migration_v39_add_budget_contributions.sql`。
